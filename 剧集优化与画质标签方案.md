# 剧集识别清洁 + 分集封面 + 画质标签方案 (M7)

独立于 `刮削优化方案.md`(M1-M6 已完结),面向扫库/媒体版本层的遗留问题。

- 起草:2026-04-19
- 当前阶段:M7.3 已落地
- 依赖:M1-M6 已完成的 `internal/services/scraper/` + `episode_fetch.go`
- 主要代码入口:
  - 扫库:`internal/services/scanner.go`
  - Episode 刮削:`internal/services/episode_fetch.go`
  - NameParser:`internal/services/scraper/nameparser.go`
  - 媒体版本:`internal/services/tmdb.go::applyMergedDetails` + `media_versions` 表

---

## 1. 背景与目标

M1-M6 聚焦识别率与多源合并,扫库侧仍有 3 个用户肉眼可见的问题:

1. **Episode name 被原始文件名污染**,像 `权游.S01E01.2160p.WEB-DL.x265-XXX.mkv` 被原样当作集标题。
2. **Episode 没有分集封面图**,列表视图只有 Season 海报重复。
3. **多版本无结构化画质标签**,`media_versions` 展示的仍是原始文件名,前端做不出"4K HDR"胶囊。

目标:
1. 扫库写入的 Episode name 不带文件名脏数据,让刮削能正常覆盖。
2. 扩展 `episode_fetch.go` 拉取 TMDB `still_path` 作为分集封面,扫库侧兜底读本地 `<name>-thumb.jpg`。
3. `media_versions` 增加结构化画质列(resolution/HDR/codec/source/label),前端展示胶囊。
4. **已刮削的存量库平滑升级** —— 三个幂等 backfill 任务,由 `system_config.backfill_enabled_on_startup` 开关控制启动时自动跑,admin 页同时提供手动触发。

非目标:
- 不动 Movie 合并(tmdb+studio)。
- 不做跨 Series 合并(需先补 Episode 级 `series_tmdb_id`,另起一个方案)。
- 不动刮削识别流水线。

## 2. 现状根因(已调研)

### 问题 1:Episode name = 文件名
- `scanner.go:1624` 的 `ensureCanonicalEpisodeItem`:
  ```go
  epTitle := strings.TrimSuffix(filepath.Base(primary.name), filepath.Ext(primary.name))
  ```
  直接把文件名去扩展名当标题。
- `episode_fetch.go::isEpisodePlaceholderName` 只认 `第X集` / `Episode X` / `S01E02`。文件名不是占位符 → 拒绝覆盖。
- 两端都堵:**写入就脏,刮削拉来的 TMDB 真实标题永远进不来**。

### 问题 2:Episode 无封面图
- `ensureCanonicalEpisodeItem` 的 INSERT SQL 没有 `primary_image_path / primary_image_tag` 字段(写入时就没尝试过)。
- `scrapeEpisodeMetadata` 只抽 `name + overview`,没动 TMDB episode 的 `still_path`。
- 扫库侧也没有 `<basename>-thumb.jpg` 的本地兜底。

### 问题 3:画质版本无结构化标签
- `NameParser`(`scraper/nameparser.go`)已经识别 `1080p / 2160p / HDR / DV / REMUX / x265` 这些 token,但都归入 `ParsedName.Junk`,不结构化返回。
- `media_versions.name = 原始文件名`,前端展示混乱。
- `media_versions` 表缺 resolution/hdr/codec 列。

## 3. 总体设计

```
新扫 Episode(M7.1 + M7.2):
  NameParser → epTitle = "Episode N" 占位符(S0 用 "Special N")
  primary_image_path = 本地 <basename>-thumb.jpg 若存在
  media_versions 同步写 resolution/hdr/codec/source/quality_label

刮削完 Series(M7.2):
  scrapeEpisodeMetadata 扩展:
    /tv/{tmdb}/season/{n} → episodes[].{name, overview, still_path}
    覆盖 name(仅占位符) / overview(仅空) / primary_image_path(仅空)

media_versions 写入(M7.3):
  mediainfo(Width/Height/VideoRange)优先 → 文件名 NameParser QualityTags 兜底
  quality_label = "4K HDR BluRay x265" 拼接

存量数据(M7.Backfill):
  A: Episode 标题清洗 + 触发 episode_fetch
  B: Episode 缩略图补全
  C: media_versions 画质标签回填
```

## 6. M7.3 media_versions 画质标签

### 6.1 Schema
新 migration `030_media_version_quality.sql`:
```sql
ALTER TABLE media_versions ADD COLUMN IF NOT EXISTS resolution     VARCHAR(16);
ALTER TABLE media_versions ADD COLUMN IF NOT EXISTS hdr_format     VARCHAR(16);
ALTER TABLE media_versions ADD COLUMN IF NOT EXISTS video_codec    VARCHAR(16);
ALTER TABLE media_versions ADD COLUMN IF NOT EXISTS audio_codec    VARCHAR(16);
ALTER TABLE media_versions ADD COLUMN IF NOT EXISTS source         VARCHAR(16);
ALTER TABLE media_versions ADD COLUMN IF NOT EXISTS quality_label  VARCHAR(64);
CREATE INDEX IF NOT EXISTS idx_media_versions_resolution
    ON media_versions (resolution) WHERE resolution IS NOT NULL;
```

### 6.2 NameParser QualityTags
`scraper/nameparser.go` 的 `ParsedName` 扩:
```go
type QualityTags struct {
    Resolution string  // "4k" / "1080p" / "720p" / "sd" / ""
    HDRFormat  string  // "hdr10" / "hdr10+" / "dv" / "sdr" / ""
    VideoCodec string  // "x265" / "x264" / "av1" / ""
    AudioCodec string  // "atmos" / "truehd" / "dts-hd" / "ac3" / ""
    Source     string  // "bluray" / "web-dl" / "remux" / "hdtv" / "dvdrip" / ""
}

type ParsedName struct {
    // ... 原字段
    Quality QualityTags
}
```

改 `stripNoise`:识别到 token 归一到对应字段,同时**保留**写入 Junk(日志兼容):
- `2160p/4k/8k` → `"4k"`(8k 稀有,按 4k 聚合也可接受;若要区分可单独 `"8k"`)
- `1080p/1440p` → `"1080p"` / `"1440p"`
- `hdr10\+?` → `"hdr10"` / `"hdr10+"`
- `dv / dolby\.?vision` → `"dv"`
- `x265 / hevc / h265 / h\.265` → `"x265"`
- `bluray / bdrip / bdremux / blu-ray` → `"bluray"`
- `web-?dl / webrip` → `"web-dl"` / `"webrip"`
- `remux` 优先级最高,若同时有 `bluray + remux` 取 `"remux"`
- `atmos` 优先于 `truehd / dts-hd`(更高阶)

### 6.3 抽取器
新文件 `internal/services/scraper/quality_tags.go`:
```go
func QualityFromMediainfo(mi map[string]any) QualityTags
func QualityFromParsed(p ParsedName) QualityTags
func MergeQualityTags(primary, fallback QualityTags) QualityTags  // 按字段补空
func QualityLabel(t QualityTags) string                           // 拼 "4K HDR BluRay x265"
```

`QualityFromMediainfo` 规则:
- `Width/Height` 推 resolution:Height ≥ 2160 → `"4k"`、≥ 1440 → `"1440p"`、≥ 1080 → `"1080p"`、≥ 720 → `"720p"`、其它 → `"sd"`
- `VideoRange / HDR_Format` 推 hdr_format
- `CodecID / Format` 推 video_codec

`QualityLabel` 顺序:resolution → hdr → source → video_codec → audio_codec(仅取不重复的前 2-3 段)。

### 6.4 写入
`scanner.go::ensureEpisodeMediaVersions` 和 Movie insert 路径:
```go
miQuality := scraper.QualityFromMediainfo(mi)
nameQuality := scraper.QualityFromParsed(scraper.Parse(f.name, scraper.ModeEpisode))
q := scraper.MergeQualityTags(miQuality, nameQuality)
label := scraper.QualityLabel(q)

INSERT INTO media_versions (..., resolution, hdr_format, video_codec, audio_codec, source, quality_label)
VALUES (..., $N+1, $N+2, ...)
```

## 8. 存量数据回填(M7.Backfill)

### 8.1 任务框架
新文件 `internal/services/backfill_task.go`:
```go
type BackfillTask struct {
    mu         sync.Mutex
    progress   BackfillProgress
    cancel     context.CancelFunc
}

type BackfillProgress struct {
    Status       string   // "idle" / "running" / "stopping" / "completed" / "error"
    Stage        string   // "quality" / "name" / "image"
    Processed    int
    Total        int
    LastError    string
    StartedAt    time.Time
    CompletedAt  time.Time
}

func (t *BackfillTask) Start(ctx, pool, stages []string)   // stages: ["quality","name","image"] 顺序
func (t *BackfillTask) Stop()
func (t *BackfillTask) GetProgress() BackfillProgress
```
运行时顺序固定 **C → A → B**(本地最快 → 轻 API → 重 API),先让用户看到变化,再对 TMDB 施压。

### 8.2 Backfill A: Episode 标题清洗
新文件 `internal/services/backfill_episode_name.go`。

**检测条件**(只改脏,不误杀用户编辑过的标题):
```sql
SELECT id, name, index_number, season_id
FROM items
WHERE type = 'Episode'
  AND index_number IS NOT NULL
  AND name IS NOT NULL
  AND NOT (name ~ '^Episode [0-9]+$' OR name ~ '^Special [0-9]+$' OR name ~ '^第[0-9]+[集话]$')
  AND (
       name ~* '\.(mkv|mp4|avi|ts|m2ts|rmvb|flv|mov|wmv)$'
    OR name ~* '(1080p|2160p|720p|480p|4k|webrip|web-?dl|bluray|bdrip|hdtv|dvdrip|remux|x26[45]|hevc|atmos|truehd|dts-?hd)'
    OR name ~ '\[.+\]\[.+\]'       -- 多段方括号(发布组惯用)
    OR name ~ '[._]{2,}'            -- 连续下划线/点号
  )
```

**处理**:
1. 每条改 `name = 'Episode ' || index_number`
2. 聚合 `DISTINCT series_id`,每个 Series 查 `tmdb_id`,对有 tmdb_id 的 Series 调 `scrapeEpisodeMetadata` 覆盖(幂等)
3. 进度按"Episode 条数"计

### 8.3 Backfill B: Episode 缩略图
新文件 `internal/services/backfill_episode_image.go`。

**检测**:
```sql
SELECT e.id, e.index_number, e.season_id, se.tmdb_id, pa.tmdb_id AS series_tmdb_id
FROM items e
JOIN items se ON se.id = e.season_id
JOIN items sr ON sr.id = se.parent_id
WHERE e.type = 'Episode'
  AND e.primary_image_path IS NULL
  AND sr.tmdb_id IS NOT NULL AND sr.tmdb_id > 0
```

**处理**:
1. 按 `series_tmdb_id + season_number` 聚合(一次 API 调用拿整季)
2. 先尝试本地 `<basename>-thumb.jpg` 兜底
3. 本地无则从 `episodes[].still_path` 下载
4. 限速:每季 API 后 200ms,每 Series 之间 500ms
5. 可被 `BackfillTask.Stop` 打断

### 8.4 Backfill C: media_versions 画质标签
新文件 `internal/services/backfill_media_quality.go`。

**检测**:
```sql
SELECT id, name, file_path, mediainfo
FROM media_versions
WHERE resolution IS NULL
```

**处理**(纯本地):
1. 先 `QualityFromMediainfo(mi)`
2. 缺的字段用 `QualityFromParsed(scraper.Parse(name, ModeEpisode))` 补
3. `UPDATE media_versions SET resolution=$1, hdr_format=$2, ...`
4. 批量 100 条一个 tx,无需外部 API,速度几秒到几十秒

### 8.5 启动自动触发
`main.go` 启动流程,在迁移执行完之后:
```go
if enabled := readBool(pool, "backfill_enabled_on_startup", false); enabled {
    go func() {
        backfillTask.Start(ctx, pool, []string{"quality", "name", "image"})
    }()
}
```

**防重复**:
- `backfill_last_run_at` 记录上次完成时间
- 启动检查:若距离 `backfill_last_run_at` < 24h,跳过自动触发
- 手动触发不看该限制

### 8.6 手动触发(admin 页)
新 handlers:
- `POST /Library/Backfill/Start` body `{ stages: ["quality", "name", "image"] }`
- `GET /Library/Backfill/Progress`
- `POST /Library/Backfill/Stop`

前端 `MetadataPage.vue` 新增"历史数据回填"卡片:
- 开关:启动时自动回填(写 `backfill_enabled_on_startup`)
- 3 个按钮:
  - 回填画质标签(快)
  - 清洗 Episode 标题(中,需 TMDB)
  - 补分集缩略图(慢,需 TMDB)
- 总按钮:全部执行(按 C→A→B 顺序)
- 进度条 + 当前阶段 + 已处理/总数
- 上次完成时间显示

### 8.7 幂等性保证
- A:只改含脏特征的 name,改后已是占位符,再跑不命中
- B:only `primary_image_path IS NULL`,写入后不命中
- C:only `resolution IS NULL`,写入后不命中

**强制重刷入口**(admin 页隐藏选项):
- "重置画质标签字段" → `UPDATE media_versions SET resolution=NULL, hdr_format=NULL, ...`
- "重置 Episode 封面" → `UPDATE items SET primary_image_path=NULL WHERE type='Episode' AND primary_image_path LIKE 'data/metadata/%/still.jpg'`
- 之后再点手动回填即可

## 12. 风险与缓解

- **Backfill A 误改真标题**:严格脏数据正则(文件扩展名 / 画质 token / 多段方括号 / 连续 `._`);模糊标题不动。提供"停止"按钮。
- **Backfill B 的 TMDB 配额**:按季聚合调用(一部剧一季一次),限速 + 随时停,受 `episode_still_fetch` 总开关控制。
- **mediainfo 缺失**(特别是 strm):NameParser 兜底;极端情况没任何标签,label 为空字符串,前端不展示胶囊。
- **用户关心进度**:三个 backfill 都实时写 `BackfillProgress`,admin 页展示。
- **占位符覆盖后手动编辑被重写**:`isEpisodePlaceholderName` 的判定严格,用户编辑后的标题不会被再覆盖。

## M7.3 落地摘要(2026-04-19)

### 产物
