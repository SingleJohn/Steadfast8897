# 刮削优化与多源接入方案

面向后续 agent 的长期规划文档。状态字段会随落地进度更新。

- 起草：2026-04-17
- 当前阶段：M6-A + M6-B 已落地,整套规划完成
- 主要代码入口：`internal/services/tmdb.go`、`internal/services/scanner.go`、`internal/services/scraper/`

---

## 1. 背景与目标

当前刮削全部走 TMDB 单源，识别率低、中文/动画覆盖差，失败项无持久化反馈。用户希望：

1. 显著提升扫库刮削的识别率；
2. 接入更多刮削源（TMDB 之外）；
3. 保留"自动刮削"为默认行为，同时提供"人工确认"开关；
4. 字段级合并策略先写死默认值，后期再做 UI。

非目标：本轮不改播放链路、不改 Emby 兼容层、不动 merge_versions 逻辑。

---

## 2. 现状问题速记

读 `tmdb.go` + `scanner.go` 后总结（位置写了行号，后续 agent 可直接定位）：

1. **文件名解析粗糙**（`scanner.go:344-402`）
   - `movieRE1/2/3` 只识别三种基本格式；`S01E01 / 1x01 / E01` 三条 episode 正则覆盖不全。
   - 无噪声剥离（分辨率、编码、压制组、HDR/DV/REMUX、`[xxx]` 标签等都会污染查询 query）。
   - `extractShowNameFromEpisodes`（`scanner.go:615`）只靠 `epRE1` 切字符串，动画命名（`[Sakura][XXX][01][1080p]`）完全失效。
2. **搜索是"一次取首条"**（`tmdb.go:504-537`、`tmdb.go:327-364`）
   - 不打分、不对比年份/IMDb/原名；失败后无降级。
   - `itemType` 在扫库阶段被钉死 `Movie/Series`，分错就永远搜不到（如纪录片、动画电影）。
3. **强识别信号被浪费**
   - NFO 的 `<imdbid>/<tmdbid>` 已入库（`scanner.go:237-242`），但 `ScrapeItemWithClient`（`tmdb.go:867-880`）仍走按名搜，不优先走 ID 直达。
   - 文件名 / 目录名内嵌的 `{tmdb-123}`、`tt1234567`、`{tvdb-456}` 没有提取。
4. **中文/动画天然劣势**
   - 默认 `zh-CN` 搜索但 TMDB 中文索引稀疏；英文原名没被抽取作为 fallback。
   - 豆瓣只用于补 `overview`（`tmdb.go:963`），未参与识别阶段。
5. **无失败缓存与重跑控制**
   - `platform_scan_state` 只记刮削阶段结果，不区分"识别失败"和"识别成功但无平台"，也没有冷却时间，重试时会把同一批失败项再次耗掉 TMDB 额度。

## 3. 总体架构（目标态）

```
Scanner 入库
   │  raw dir/file name
   ▼
┌──────────────┐
│ NameParser   │  输出 {Title, OriginalTitle, Year, Season, Episode, IDs, MediaHint, Junk}
└──────┬───────┘
       ▼
┌──────────────┐      强信号短路
│ Matcher      │◄───── IDs 命中 → 直达 Provider.GetByID
│ (识别)       │
│              │      弱信号走多源搜索
│   Aggregator │──► 各 Provider.Search → 候选打分 → 多源互投
└──────┬───────┘
       ▼ 命中 (>=阈值 或 人工确认)
┌──────────────┐
│ FieldMerger  │  字段级合并：overview/rating/actors/genre/poster 各走优先级策略
└──────┬───────┘
       ▼
   写回 items + item_external_ids + 图片下载 + NFO 落盘（原逻辑保留）
```

关键抽象：`NameParser`、`Matcher`、`Provider`、`Aggregator`、`FieldMerger`。

## 4. 识别率优化（M1 - M2，不新增刮削源）

### M1 NameParser

新增 `internal/services/scraper/nameparser.go`（包名 `scraper`，为后续拆包铺路）。

**输出结构**
```go
type ParsedName struct {
    Title         string              // 归一化主标题（去噪声后）
    OriginalTitle string              // 英文原名（如能分离）
    Year          *int32
    Season        *int32
    Episode       *int32
    IDs           map[string]string   // "tmdb"/"imdb"/"tvdb"/"bangumi"/"douban"
    MediaHint     string              // "anime"/"movie"/"series"/"" — 纯启发式
    Junk          []string            // 被剥掉的 token，用于日志和回溯
}
```

**规则优先级（从高到低，命中即停）**

1. 显式 ID：`{tmdb-123}`、`{imdb-tt123}`、`{tvdb-456}`、裸 `tt\d{7,8}`、`{bgm-789}`。
2. 季数：`第X季`、`Season X`、`S0X`（但不吃 `S01E02` 里的 S01 给 Season 字段，走 episode 规则优先）。
3. Episode：`S01E02`、`1x02`、`EP02`、`- 02`、`[02]`、`第02[话集]`、`#02`。
4. 噪声剥离白名单（正则集中维护）：
   - 分辨率：`720p|1080p|1440p|2160p|4K|8K`
   - 源：`BluRay|BDRip|WEB-?DL|WEBRip|HDTV|DVDRip|REMUX`
   - 编码：`x26[45]|h\.?26[45]|HEVC|AVC|AV1`
   - 音轨：`DTS(-HD)?|TrueHD|Atmos|AC3|EAC3|FLAC|AAC|[57]\.1`
   - HDR：`HDR(10\+)?|DV|Dolby.?Vision|SDR`
   - 其他：`CHS|CHT|GB|BIG5|简[繁]|繁体|国语|粤语|双语`、Scene 组后缀 `-[A-Z0-9]+$`、`[发布组]` 方括号块。
5. 多语言拆标题：
   - `中文名.English Name.Year` → Title = 中文名, OriginalTitle = English Name。
   - `中文名 English Name` → 同上，按首个 ASCII 连续片段切。
6. 剩余主串作为 `Title`，替换 `._` 为空格并 trim。

**MediaHint 启发**
- 路径含 `动画/番剧/Anime` 或文件名有方括号发布组 → `anime`。
- 有 `S\d+E\d+` / `第X季` → `series`。
- 仅 Year，无季集 → `movie`。

**测试要求**：对用户能提供的真实难例建 table-driven 测试（不由 Claude 运行；写好后用户本地跑 `go test ./internal/services/scraper`）。

**替换点**
- `scanner.go:ParseMovieName` → `scraper.Parse(name, ctx)`
- `scanner.go:ParseEpisodeInfo` → 从 `ParsedName.Season/Episode` 取
- `scanner.go:extractShowNameFromEpisodes` → 改为遍历文件调用 `scraper.Parse`，按 `Title` 众数选剧名

### M2 Matcher + ID 直达 + 缓存

新增 `internal/services/scraper/matcher.go`。

**流水线（TMDB 单源版，Provider 接口落地前的过渡实现）**

1. 若 `ParsedName.IDs["tmdb"] != ""` → `GetMovieDetails/GetTVDetails` 直达，置信度 1.0。
2. 若 `ParsedName.IDs["imdb"] != ""` → `GET /find/tt...?external_source=imdb_id`，拿 tmdb id 后直达。
3. `Search(OriginalTitle, Year)` → 取 top5，按以下打分：
   ```
   score = 0.55 * titleSim + 0.25 * yearMatch + 0.15 * origTitleSim + 0.05 * popularity
   titleSim:     Jaro-Winkler on lower(NFD-normalized)
   yearMatch:    1 if diff==0 else 0.5 if diff<=1 else 0
   popularity:   min(1.0, candidate.popularity / 50)
   ```
   阈值 ≥ 0.72 采纳。
4. 降级链：
   - 去 Year 再搜
   - 改用 Title（中文）再搜
   - `search/multi`（同时搜电影/剧）
   - 仅保留标题首个连续块（剥所有空格后标记）
5. 全失败 → 写 `platform_scan_state` with 新 `unidentified` 状态 + reason；不再强塞低置信结果。

**ID 优先捷径**
- `ScrapeItemWithClient` 进入前，先读 `items.imdb_id / tmdb_id` 和同目录 NFO 的 ID → 命中即走分支 1/2。

**Redis 缓存**
- Key：`scraper:search:{provider}:{type}:{sha1(query|year)}`
- Value：`{candidates_json, ts}`，TTL 7 天（失败结果 TTL 24h，避免坏缓存长存）。
- 由 `CacheService` 扩展新方法，不直接裸调 Redis。

**数据库变更**
- 新 migration（序号按落地时最大 +1，当前最大是 `026`，预计 `027_platform_scan_unidentified.sql`）：
  ```sql
  -- 扩展 platform_scan_status 枚举值：'unidentified' 表示识别阶段就失败
  ALTER TABLE items ADD COLUMN IF NOT EXISTS identify_attempted_at TIMESTAMPTZ;
  ALTER TABLE items ADD COLUMN IF NOT EXISTS identify_cooldown_until TIMESTAMPTZ;
  CREATE INDEX IF NOT EXISTS idx_items_identify_cooldown
      ON items (identify_cooldown_until) WHERE identify_cooldown_until IS NOT NULL;
  ```
  重试时跳过 `identify_cooldown_until > NOW()` 的 item。失败后冷却 24h；手动触发可绕过。

### M4.5 补漏摘要（2026-04-19）

M4 审查发现三处关键偏离需要先补齐,否则 M5 的未匹配面板 / 字段优先级 UI 建在浮砂上:
- **Aggregator.Identify "首个命中即返回"**(`aggregator.go:94-106`),不是方案 §5.3 的多源投票。
- **Aggregator.Fill 只调命中 provider 的 `GetByID`**(`fill.go:13`),豆瓣/Bangumi 数据实际上永远不会进库。

产物:

- 新文件 `internal/services/scraper/ratelimit.go`:`Gate{rps, burst, minGap, window, breaker}` — 不引入 `golang.org/x/time/rate` 依赖(go.mod 未声明,CLAUDE 规范禁止运行 `go get`),自实现 token bucket + 5 分钟滑窗错误率熔断。`Gate.Wait(ctx)` 限流阻塞,`Gate.Observe(success)` 反馈;达到错误率阈值时 `Wait` 返回 `ErrBreakerOpen`,Cooldown 到期自动半开。

### 5.1 Provider 抽象

新增目录 `internal/services/scraper/providers/`，每个源一个文件：`tmdb.go`、`bangumi.go`、`douban.go`、`tvdb.go`、`fanart.go`。

```go
// internal/services/scraper/provider.go
type MediaType string
const (
    MediaMovie  MediaType = "Movie"
    MediaSeries MediaType = "Series"
)

type Query struct {
    Title         string
    OriginalTitle string
    Year          *int32
    Language      string
    Hint          string       // "anime" 等
}

type Candidate struct {
    ProviderID    string       // e.g. "tmdb:123"
    Title         string
    OriginalTitle string
    Year          *int32
    Overview      string
    PosterURL     string
    Popularity    float64
    ExternalIDs   map[string]string
    Raw           map[string]interface{}
}

type Details struct {
    IDs             map[string]string
    Title, OriginalTitle, Overview, Tagline string
    Year            *int32
    Premiered       *string
    Rating          *float64
    Genres          []string
    Studios         []string
    Actors          []NfoActor
    Directors       []string
    PosterURLs      []string
    BackdropURLs    []string
    SeasonPosters   map[int32]string
    Episodes        []EpisodeMeta
}

type Provider interface {
    Name() string                 // "tmdb"/"bangumi"/"douban"/"tvdb"/"fanart"
    Priority() int                // 小数字优先；冲突时作为 tie-breaker
    Supports(MediaType) bool
    Search(ctx, MediaType, Query) ([]Candidate, error)
    GetByID(ctx, id string) (*Details, error)
    FindByExternalID(ctx, kind, id string) (*Details, error)  // 可选，实现返回 ErrUnsupported
}
```

### 5.3 Aggregator

```go
// internal/services/scraper/aggregator.go
func (a *Aggregator) Identify(ctx, parsed ParsedName, hint MediaType) (*Match, error)
func (a *Aggregator) Fill(ctx, itemID, match Match) error
```

**识别阶段**
- 并发调用各启用 Provider 的 `Search`；IDs 命中时并发 `FindByExternalID` / `GetByID`。
- 对所有候选构造 `unified_key = 主 provider 的 external_ids` 或 fallback `(title_norm, year)` → 聚合打分：
  ```
  unified_score = Σ (provider_weight * local_score)   (归一化后)
  ```
- 多源互投：≥2 个源指向同一 unified_key → 直接采纳；否则取 top1 若 > 阈值。
- 仍失败 → `unidentified`。

**填充阶段（字段级合并，默认策略写死在 `field_policy.go`）**

```
overview:       tmdb(zh) → douban → bangumi → tmdb(en)
title:          tmdb(zh) → douban → bangumi → tmdb(en)
original_title: tmdb(en) → imdb → tmdb(zh)
rating:         douban → tmdb → bangumi        (取首个非 null)
premiered/year: tmdb → bangumi → douban
genres:         union，走 CanonicalPlatformName 类似的归一
actors:         tmdb（有图片）主，douban 补中文名（按 name 精确或 Jaro>0.9 匹配）
directors:      union
studios/platform: 已有 ExtractPlatform，沿用并接受其他源输入
poster:         tvdb(series) → tmdb → fanart
backdrop:       fanart → tmdb → tvdb
season_poster:  tvdb → tmdb → fanart
```

## 6. 人工确认开关（贯穿 M2 - M4）

- `scrape_auto_apply = true`（默认）：低于阈值的结果仍然走"最佳候选自动采纳"，只记日志；用户仍可在"待匹配"页重新指定。
- `scrape_auto_apply = false`：低于阈值不写入 `items`，进入 `identify_candidates` 表，前端显式采纳或选择其他候选。

新 migration（与 external_ids 同批，或编号连续）：

```sql
CREATE TABLE IF NOT EXISTS identify_candidates (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    item_id    UUID NOT NULL REFERENCES items(id) ON DELETE CASCADE,
    provider   TEXT NOT NULL,
    external_id TEXT NOT NULL,
    title      TEXT,
    year       INT,
    poster_url TEXT,
    score      REAL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_identify_candidates_item
    ON identify_candidates (item_id);
```

前端复用现有 `ScrapeItemByTMDBID`（`tmdb.go:831`）作为采纳接口；仅需新增 candidates 列表 API。
